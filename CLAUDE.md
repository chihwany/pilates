# Pilates AI - 프로젝트 규칙

## 프로젝트 개요
AI 기반 필라테스 맞춤 운동 시퀀스 생성 앱. 회원이 카메라로 얼굴 촬영 → Claude Vision 컨디션 분석 → 회원 결과 수정 → LLM 시퀀스 자동 생성 → 강사 선택적 리뷰.

## 핵심 설계 원칙
- **얼굴 인식/본인 확인 없음**: 로그인 기반 사용자 식별. 카메라는 컨디션 분석 전용.
- **얼굴 사진 등록 없음**: 회원가입 시 얼굴 사진 불필요. 생체 데이터 저장 최소화.
- **촬영 사진 미보관**: 분석 후 서버에 이미지 저장하지 않음. 분석 결과(JSON)만 DB 저장.
- **AI 원본 + 회원 수정본 이중 저장**: `conditionAiRaw`(AI 원본) + `conditionFinal`(회원 수정 후).
- **단일 AI 서비스**: Anthropic API만 사용 (Claude Vision + Claude Text). AWS Rekognition 등 미사용.
- **자동 배정**: 시퀀스 생성 즉시 회원에게 배정. 강사 확정 과정 없음(선택적 리뷰만).

## 기술 스택
- **Frontend**: React Native, Expo (SDK 54), expo-router v4, NativeWind v4, Zustand v5, TanStack Query v5, expo-camera (CameraView), react-native-gesture-handler, react-native-reanimated, expo-notifications
- **Backend**: Node.js 20, Hono v4, Drizzle ORM, PostgreSQL 15+ (Docker), bcrypt, jsonwebtoken, node-cron, expo-server-sdk, @anthropic-ai/sdk
- **AI**: Claude Vision API (컨디션 분석, claude-sonnet-4-20250514, tool_use 패턴), Claude Text API Sonnet (시퀀스 생성)
- **Infra**: Docker (PostgreSQL), Railway (서버, 프로덕션), Expo EAS (빌드), Expo Push Service (알림)
- **인증**: 자체 JWT (bcrypt + jsonwebtoken). Supabase 미사용.

## 프로젝트 구조
```
pilates/
├── front/                # Expo 프론트엔드
│   ├── app/              # Expo Router 파일 기반 라우트
│   │   ├── (auth)/       # 로그인, 회원가입
│   │   ├── (instructor)/ # 강사 탭: 대시보드, 회원, 스케줄, 세션, 설정
│   │   └── (member)/     # 회원 탭: 오늘, 컨디션, 기록, 프로필
│   ├── components/       # 재사용 컴포넌트 (ui/, camera/NativeCamera+WebCamera, member/, condition/, schedule/, exercise/)
│   ├── lib/              # API 클라이언트 (auth, client, condition, members, schedules, sequences), stores, hooks, types, utils
│   ├── shared/           # 프론트/백 공유 타입
│   └── assets/           # 정적 자원
├── server/               # Hono 백엔드
│   ├── src/routes/       # API 라우트 (auth, members, schedules, condition, sequences, exercises)
│   ├── src/services/     # claude-vision.ts (@anthropic-ai/sdk), sequence-generator.ts
│   ├── src/middleware/    # JWT 인증, 역할 검증
│   ├── src/lib/          # JWT, password 유틸
│   ├── src/jobs/         # 크론잡 (12시 알림, 1시 자동생성)
│   └── src/db/           # Drizzle 스키마, 마이그레이션
└── docs/                 # 프로젝트 설계 문서 + 에이전트 가이드 + 스킬 레퍼런스
```

## 실행 방법
```bash
# 백엔드 (포트 3000)
cd server && npm run dev

# 프론트엔드 - 웹 (포트 8081)
cd front && npx expo start --web

# 프론트엔드 - 모바일
cd front && npx expo start
```

## 환경변수
- `server/.env`에 `ANTHROPIC_API_KEY` 필요 (Claude Vision/Text API)
- API 키 없거나 실패 시 mock fallback으로 동작

## 인증 플로우 상세
- API client: 401 응답 시 자동 logout 하지 않음 (로그인 루프 방지)
- authStore loadToken: 저장된 토큰을 `/auth/me`로 유효성 검증, 만료 시 자동 삭제
- 오늘 탭: 401 응답 시 에러 대신 빈 상태(시퀀스 없음)로 표시
- CORS: 모든 origin 허용 (모바일 개발 환경 호환)

## 카메라 컴포넌트
- **NativeCamera** (`front/components/camera/NativeCamera.tsx`): expo-camera CameraView, 전면 카메라, base64 촬영, 권한 요청 UI
- **WebCamera** (`front/components/camera/WebCamera.tsx`): 브라우저 getUserMedia API, 웹캠 캡처, 촬영 이미지 미리보기

## 슬라이더 UX
- 웹: HTML input range (스무스 조작)
- 네이티브: react-native-gesture-handler + react-native-reanimated (스무스 조작)
- 에너지 슬라이더: "나쁨" ~ "좋음", 색상 반전 (낮으면 빨강, 높으면 초록)
- 스트레스 슬라이더: "적음" ~ "많음" (낮으면 초록, 높으면 빨강)
- 값 라벨: 숫자 대신 "매우 낮음/낮음/보통/높음/매우 높음" 텍스트

## 주요 데이터 모델
- `users`: 인증 (email, passwordHash, role, pushToken)
- `members`: 회원 프로필 (bodyConditions, exercisePreferences, fitnessLevel)
- `weeklySchedules`: 주간 수업 스케줄 (요일, 시간, 강사-회원 매핑)
- `sessions`: 세션 (conditionAiRaw, conditionFinal, memberNote, requestedCategories, isAutoGenerated)
- `exerciseSequences`: 운동 시퀀스 (exercises JSONB 배열)
- `exerciseCatalog`: 운동 카탈로그 (참조 데이터)

## API 엔드포인트 규칙
- 모든 엔드포인트는 `/api/` 접두사 사용
- 인증 필요: `Authorization: Bearer {JWT}`
- 역할 구분: 강사 전용 엔드포인트는 `roleGuard('instructor')` 미들웨어
- 에러 응답: `{ error: { code, message, statusCode } }` 형식 통일

## 컨디션 분석 항목
기본: 에너지(1-10), 무드(HAPPY/CALM/SAD/STRESSED/TIRED), 스트레스(1-10), 컨디션요약
추가: 수면상태(GOOD/FAIR/POOR), 안면근긴장(이마/턱/비대칭), 부종(NONE/MILD/MODERATE), 피부(창백/홍조)
각 항목에 confidence score 포함. 낮은 신뢰도 항목은 시퀀스에 가볍게만 반영.
