# PRD: Pilates AI - 맞춤 운동 시퀀스 생성 앱

## 1. 개요

### 1.1 프로젝트 명
**Pilates AI** - AI 기반 필라테스 맞춤 운동 시퀀스 생성 앱

### 1.2 문제 정의
필라테스 강사는 매 수업 전 회원의 현재 컨디션(피로, 스트레스, 기분)을 파악하고, 기존 신체 상태(측만증, 디스크 등)와 운동 선호도를 고려하여 수동으로 운동 프로그램을 구성한다. 이 과정은 시간이 많이 걸리고, 강사의 경험과 기억에 의존하여 일관성이 떨어질 수 있다.

### 1.3 솔루션
회원이 앱에 로그인한 후 카메라로 자신의 얼굴을 촬영하면, Claude Vision AI가 감정/컨디션을 즉시 분석한다. 회원은 분석 결과를 확인하고 필요시 수정한 뒤 등록한다. 이 컨디션 정보와 회원의 기본 프로필(신체 상태, 운동 선호도, 피트니스 레벨)을 결합하여 LLM이 오늘의 맞춤 운동 시퀀스를 자동 생성하고 즉시 배정한다. 강사는 필요 시에만 시퀀스를 리뷰하고 수정할 수 있다.

> **참고**: 로그인 상태에서 촬영하므로 별도의 얼굴 인식/본인 확인 과정은 불필요하다. 얼굴 사진 사전 등록도 필요 없다.

### 1.4 목표 사용자
| 사용자 | 역할 |
|---|---|
| **필라테스 강사** | 회원 관리(등록/편집), 주간 수업 스케줄 관리, 자동 생성된 시퀀스 리뷰/수정, 세션 현황 관리 |
| **회원** | 카메라 컨디션 체크, 분석 결과 확인/수정, 추가 운동 카테고리 선택, 오늘의 시퀀스 확인, 운동 기록 조회 |

## 2. 핵심 기능

### 2.1 카메라 기반 컨디션 분석
- 회원이 로그인 후 카메라로 얼굴을 촬영
  - 네이티브: NativeCamera 컴포넌트 (expo-camera CameraView, 전면 카메라, base64 촬영, 권한 요청 UI)
  - 웹: WebCamera 컴포넌트 (브라우저 getUserMedia 웹캠, 촬영 후 이미지 미리보기)
- Claude Vision API(claude-sonnet-4-20250514, tool_use 패턴)가 촬영된 사진에서 다음 항목을 분석:

**기본 분석 항목:**
| 항목 | 설명 | 시퀀스 반영 방법 |
|---|---|---|
| **에너지 레벨** (1~10) | 표정 활력, 눈의 생기 종합 | 낮으면 회복/스트레칭 위주, 높으면 챌린지 강화 |
| **무드** | HAPPY, CALM, SAD, STRESSED, TIRED | STRESSED → 호흡 중심, HAPPY → 활동적 운동 |
| **스트레스 지표** (1~10) | 긴장/불안 관련 표정 종합 | 높으면 이완/마인드풀니스 운동 비중 증가 |
| **컨디션 요약** | 자연어 기반 상세 설명 | LLM 시퀀스 생성 시 컨텍스트로 활용 |

**추가 분석 항목 (얼굴에서 읽을 수 있는 운동 관련 정보):**
| 항목 | 분석 포인트 | 시퀀스 반영 방법 |
|---|---|---|
| **수면 상태** (GOOD/FAIR/POOR) | 다크서클, 눈 부기, 눈 개방도 종합 | POOR → 고강도 제외, 부드러운 회복 시퀀스 |
| **안면 근긴장** | 이마 주름/미간 찡그림, 턱 긴장(이 악물기), 좌우 비대칭 | 이마 긴장 → 경추 스트레칭 추가, 턱 긴장 → 목-어깨 이완 강화 |
| **안면 부종** (NONE/MILD/MODERATE) | 얼굴 윤곽 팽만감, 볼/턱선 부기 | 부종 → 림프 순환 촉진 워밍업 (목/어깨 스트레칭) 추가 |
| **피부 상태** | 창백함/홍조 | 보조 지표: 창백 → 급격한 체위 변화 제한, 홍조 → 쿨다운 강화 |

> **신뢰도 기반 반영**: 조명/화장 등의 영향으로 분석 신뢰도가 낮은 항목은 시퀀스에 가볍게만 반영된다. 각 항목에 confidence score를 포함하여, 확실한 경우만 적극 반영한다.

### 2.2 컨디션 결과 확인 및 수정
- AI 분석 결과를 회원에게 보여주고, **회원이 직접 수정할 수 있는 기능** 제공
- 수정 가능 항목:
  - 에너지 레벨 (슬라이더 조절, "나쁨" ~ "좋음", 낮으면 빨강/높으면 초록)
  - 무드 (선택 변경)
  - 스트레스 지표 (슬라이더 조절, "적음" ~ "많음", 낮으면 초록/높으면 빨강)
  - 수면 상태 (선택 변경)
  - 추가 메모 (자유 텍스트, 예: "오른쪽 어깨가 좀 뻐근해요")
- 슬라이더 UX: 값 라벨은 숫자 대신 "매우 낮음/낮음/보통/높음/매우 높음" 텍스트로 표시
  - 웹: HTML input range (스무스 조작)
  - 네이티브: react-native-gesture-handler + react-native-reanimated (스무스 조작)
- 수정 후 [등록] 버튼으로 최종 컨디션 확정
- AI 원본 분석과 회원 수정본이 모두 저장됨 (강사 참고용)

### 2.3 회원 프로필 관리
- **기본 정보**: 이름, 연락처, 생년월일
- **신체 상태**: 측만증, 후만증, 디스크, 무릎 문제 등 (구조화된 태그 + 심각도)
- **운동 선호도**: 선호 기구(리포머, 매트, 체어 등), 피하고 싶은 운동, 운동 목표
- **피트니스 레벨**: 초급 / 중급 / 고급

### 2.4 주간 수업 스케줄 관리
- 강사가 회원별 주간 수업 스케줄을 등록/관리
  - 요일별 수업 시간 설정 (예: 월/수/금 10:00)
  - 회원별 담당 강사 배정
- 스케줄 기반으로 오늘 수업 대상 회원 자동 결정
- 스케줄이 등록된 회원만 오후 1시 자동 시퀀스 생성 대상

### 2.5 AI 운동 시퀀스 생성
- 회원 프로필 + 오늘 컨디션 + 추가 운동 요청 + 최근 운동 이력을 종합하여 LLM이 시퀀스 생성
- **추가 운동 카테고리 선택**: 회원이 컨디션 체크 후 1~2개 운동 카테고리를 선택 가능 (선택 사항)
  - 예: "상체 강화", "하체 스트레칭", "코어 집중", "유연성", "밸런스" 등
  - 선택한 카테고리가 시퀀스 생성 시 우선적으로 반영됨
- **50분 수업 기준** 시퀀스: 워밍업 3개(~7분) + 메인 8~12개(~36분) + 쿨다운 3개(~7분) = 총 50분
  - 세트 간 휴식 15초 포함
  - 동적 운동 수 조절 (50분에 맞게)
- 시퀀스 구성 요소:
  - 운동명, 카테고리(웜업/코어/상체/하체/스트레칭/쿨다운)
  - 기구, 시간, 세트/횟수, 난이도
  - 신체 집중 부위, 수정사항(modification)
  - 이 운동을 선택한 이유 설명
- **컨디션 반영 규칙**:
  - 에너지 낮음 → 부드러운/회복 위주
  - 스트레스 높음 → 호흡/마인드풀니스 강조
  - 수면 부족 → 고강도 제외, 점진적 강도 증가
  - 안면 근긴장 감지 → 해당 부위 이완 운동 추가
  - 금기 운동 자동 제외
- **컨디션 미체크 시**: 컨디션 정보를 제외한 나머지 정보(프로필, 운동 이력, 선호도)만으로 시퀀스 생성
- **오후 1시 자동 마감**: 오후 1시까지 컨디션 체크를 하지 않은 회원은 자동으로 컨디션 없이 시퀀스 생성

### 2.6 Push 알림
- **12시 컨디션 체크 리마인더**: 수업 당일 오후 12시까지 컨디션 체크를 하지 않은 회원에게 push 알림 발송
  - "오늘 수업이 있습니다! 컨디션 체크를 진행해주세요. (마감: 오후 1시)"
- **시퀀스 생성 완료 알림**: 시퀀스가 생성/배정되면 회원에게 알림
- **강사 시퀀스 수정 알림**: 강사가 시퀀스를 수정하면 해당 회원에게 알림

### 2.7 강사 리뷰 및 조정 (선택적)
- 자동 생성된 시퀀스가 즉시 회원에게 배정됨
- 강사는 대시보드에서 오늘의 세션 현황을 확인
- 필요시 시퀀스를 선택하여 리뷰:
  - 드래그 앤 드롭으로 순서 변경
  - 운동 추가/삭제/수정
- 수정 내용은 즉시 회원에게 반영

### 2.8 회원 뷰
- 오늘 배정된 시퀀스 확인
- 시퀀스 없을 때: "오늘의 컨디션을 체크해 주세요" + "컨디션 체크하러 가기" 버튼 (빈 상태 UX)
- 운동별 상세 설명 및 수정사항
- 과거 세션 히스토리

### 2.9 로그아웃
- 회원: 프로필 탭 하단 (ScrollView로 스크롤 가능)
- 강사: 설정 탭 하단

## 3. 사용자 플로우

### 3.1 회원 플로우 (핵심)
```
1. 앱 실행 → 로그인
2. [오늘] 탭 → 시퀀스 미배정 시 "컨디션 체크하기" 안내
3. [컨디션 체크] 탭 → 카메라 열림 (웹: 웹캠, 네이티브: expo-camera)
4. 얼굴 촬영 → 이미지 미리보기 (웹) → Claude Vision이 컨디션 분석
5. 단계별 로딩 화면: "컨디션 분석 중" → "컨디션 등록 중" → "시퀀스 생성 중"
6. 결과 화면: "에너지: 6/10 | 무드: 차분 | 스트레스: 3/10 | 수면: 보통"
7. (선택) AI 분석 결과 수정 (슬라이더/선택지 조절 + 추가 메모)
8. [등록] → 컨디션 확정
9. (선택) 추가 운동 카테고리 1~2개 선택 (예: "코어 집중", "유연성")
10. [완료] → 시퀀스 자동 생성 + 즉시 배정
11. 시퀀스 생성 완료 후 자동으로 [오늘] 탭 이동 → 배정된 시퀀스 확인
11. 운동 카드 탭 → 상세 설명, 수정사항
12. [기록] 탭 → 과거 세션 기록
```

### 3.2 강사 플로우
```
1. 앱 실행 → 로그인
2. [대시보드] 탭 → 오늘 세션 현황 확인
   - 컨디션 체크 완료 회원 목록
   - 시퀀스 생성 완료/대기 현황
   - 오후 1시 자동 생성 대상 회원
3. (선택) 시퀀스 카드 탭 → 리뷰 화면 진입
4. 필요시 운동 수정 (순서 변경, 운동 교체/추가/삭제)
5. [저장] → 수정 내용 즉시 반영
6. [회원] 탭 → 회원 관리 (등록/편집)
7. [스케줄] 탭 → 주간 수업 스케줄 관리
8. [세션] 탭 → 전체 세션 기록 관리
```

### 3.3 알림 및 자동 생성 플로우
```
오전 ~ 수업 전:
  1. 회원이 자발적으로 컨디션 체크 진행

12:00 (정오):
  2. 서버 크론잡: 오늘 수업 스케줄이 있지만 컨디션 미체크인 회원에게 push 알림 발송
     "오늘 수업이 있습니다! 컨디션 체크를 진행해주세요. (마감: 오후 1시)"

13:00 (오후 1시):
  3. 서버 크론잡: 오늘 수업 스케줄이 있지만 여전히 미체크인 회원 대상
  4. 컨디션 없이 프로필 + 운동 이력 기반으로 시퀀스 자동 생성
  5. 시퀀스 즉시 회원에게 배정
  6. 강사 대시보드에 자동 생성 알림 표시
```

## 4. 비기능 요구사항

### 4.1 AI/분석 방식

**Claude Vision API 단독 사용 (컨디션 분석 전용)**

카메라로 촬영한 얼굴 사진을 Claude Vision API로 전송하여 컨디션만 분석한다. 로그인 상태에서 촬영하므로 별도의 얼굴 인식이나 본인 확인은 수행하지 않는다.

- **얼굴 사진 등록 없음**: 회원가입 시 얼굴 사진을 등록할 필요가 없다.
- **컨디션 분석**: 촬영된 사진 한 장으로 감정, 에너지, 스트레스, 수면 상태, 근긴장 패턴 등을 종합 분석한다.
- **회원 수정 가능**: AI 분석 결과를 회원이 검토하고 수정할 수 있어, 분석 오차를 보완한다.
- **장점**: 단일 AI 서비스(Anthropic)로 컨디션 분석 + 시퀀스 생성을 모두 처리하여 외부 의존성이 최소화된다.

### 4.2 성능
- 컨디션 분석: 캡처 후 5초 이내 결과
- 시퀀스 생성: 10초 이내 (로딩 인디케이터 표시)

### 4.3 보안 및 개인정보
- 촬영된 얼굴 사진은 분석 후 서버에 보관하지 않음 (분석 결과만 저장)
- 모든 통신 HTTPS
- JWT 기반 인증 (15분 만료 + 리프레시 토큰)
- 역할 기반 접근 제어

### 4.4 가용성
- 오프라인 시 마지막 캐시된 시퀀스 표시
- API 장애 시 이전 시퀀스 기반 폴백

## 5. 성공 지표
| 지표 | 목표 |
|---|---|
| 컨디션 분석 정확도 (회원 수정 없이 사용 비율) | > 70% |
| 시퀀스 생성 → 강사 수정 없이 사용 비율 | > 60% |
| 수업 준비 시간 단축 | 기존 대비 50% 이상 |
| 회원 컨디션 체크 참여율 | > 70% |
