# 데이터 모델 및 API 설계 문서

## 1. 데이터 모델 (Drizzle ORM 스키마)

### 1.1 users - 사용자 (인증)

| 컬럼 | 타입 | 제약조건 | 설명 |
|---|---|---|---|
| id | uuid | PK, default: gen_random_uuid() | 사용자 고유 ID |
| email | text | UNIQUE, NOT NULL | 이메일 (로그인 ID) |
| passwordHash | text | NOT NULL | bcrypt 해싱된 비밀번호 |
| role | enum | NOT NULL | 'instructor' \| 'member' |
| name | text | NOT NULL | 이름 |
| pushToken | text | NULLABLE | Expo Push Token |
| createdAt | timestamp | default: now() | 가입일 |
| updatedAt | timestamp | default: now() | 수정일 |

> 자체 JWT 인증. 비밀번호는 bcrypt로 해싱 저장. Docker PostgreSQL에 직접 연결.

---

### 1.2 members - 회원 프로필

| 컬럼 | 타입 | 제약조건 | 설명 |
|---|---|---|---|
| id | uuid | PK | 회원 고유 ID |
| userId | uuid | FK → users.id, NULLABLE | 앱 계정 연결 (강사 직접 등록 시 null) |
| name | text | NOT NULL | 이름 |
| phone | text | NULLABLE | 연락처 |
| dateOfBirth | date | NULLABLE | 생년월일 |
| bodyConditions | jsonb | default: [] | 신체 상태 배열 |
| exercisePreferences | jsonb | default: {} | 운동 선호도 (아래 참고) |
| fitnessLevel | enum | NOT NULL, default: 'beginner' | 'beginner' \| 'intermediate' \| 'advanced' |
| notes | text | NULLABLE | 강사 메모 |
| isActive | boolean | default: true | 활성 상태 |
| createdAt | timestamp | default: now() | 등록일 |
| updatedAt | timestamp | default: now() | 수정일 |

> 얼굴 사진 등록 관련 필드 없음. 로그인 기반으로 사용자 식별.

**exercisePreferences JSONB에 포함되는 필드:**

| 키 | 타입 | 설명 |
|---|---|---|
| isPrenatal | boolean | 산전 여부 (default: false) |
| isPostnatal | boolean | 산후 여부 (default: false) |
| targetMuscles | string[] | 타겟 근육 그룹 배열 |
| avoidExercises | string[] | 피해야 할 운동 목록 |
| sessionDurationMinutes | number | 세션 시간 (default: 50, 10~120) |

> API 응답에서는 이 필드들이 최상위 레벨로 노출됩니다 (예: `data.isPrenatal`, `data.targetMuscles`).

**targetMuscles 가능한 값:**
```
core, glutes, hamstrings, quadriceps, back, shoulders, arms, chest, hip_flexors, calves, inner_thighs
```

**bodyConditions JSONB 구조:**
```json
[
  {
    "type": "scoliosis",
    "area": "lumbar",
    "severity": "mild",
    "notes": "좌측 만곡, 스트레칭 시 주의"
  }
]
```

**bodyConditions type 목록:**
```
scoliosis        - 척추측만증
kyphosis         - 척추후만증
lordosis         - 척추전만증
disc_herniation  - 디스크 탈출
spinal_stenosis  - 척추관 협착증
knee_injury      - 무릎 부상
shoulder_injury  - 어깨 부상
hip_issue        - 고관절 문제
neck_issue       - 목 문제
ankle_issue      - 발목 문제
wrist_issue      - 손목 문제
pregnancy        - 임신
postpartum       - 산후
osteoporosis     - 골다공증
arthritis        - 관절염
other            - 기타
```

**exercisePreferences JSONB 구조:**
```json
{
  "preferredEquipment": ["reformer", "mat"],
  "avoidExercises": ["rolling_like_a_ball", "open_leg_rocker"],
  "goals": ["core_strength", "flexibility", "posture_correction"],
  "sessionDurationMinutes": 50,
  "targetMuscles": ["core", "glutes", "back"]
}
```

### 1.3 weeklySchedules - 주간 수업 스케줄

| 컬럼 | 타입 | 제약조건 | 설명 |
|---|---|---|---|
| id | uuid | PK | 스케줄 고유 ID |
| memberId | uuid | FK → members.id, NOT NULL | 회원 ID |
| instructorId | uuid | FK → users.id, NOT NULL | 담당 강사 ID |
| dayOfWeek | integer | NOT NULL (0=일, 1=월 ... 6=토) | 요일 |
| startTime | time | NOT NULL | 수업 시작 시간 |
| isActive | boolean | default: true | 활성 상태 |
| createdAt | timestamp | default: now() | 등록일 |
| updatedAt | timestamp | default: now() | 수정일 |

**인덱스:**
- `memberId, dayOfWeek` UNIQUE (회원별 요일당 하나의 스케줄)
- `instructorId, dayOfWeek` (강사별 요일 스케줄 조회)
- `dayOfWeek, isActive` (오늘 수업 대상 회원 조회)

### 1.4 sessions - 세션 (수업/방문)

| 컬럼 | 타입 | 제약조건 | 설명 |
|---|---|---|---|
| id | uuid | PK | 세션 고유 ID |
| memberId | uuid | FK → members.id, NOT NULL | 회원 ID |
| instructorId | uuid | FK → users.id, NULLABLE | 강사 ID |
| date | date | NOT NULL | 수업 날짜 |
| conditionAiRaw | jsonb | NULLABLE | AI 원본 분석 결과 |
| conditionFinal | jsonb | NULLABLE | 회원 수정 후 최종 컨디션 |
| conditionCheckedAt | timestamp | NULLABLE | 컨디션 체크 시각 |
| memberNote | text | NULLABLE | 회원 추가 메모 (예: "어깨가 뻐근") |
| requestedCategories | jsonb | default: [] | 추가 운동 카테고리 |
| isAutoGenerated | boolean | default: false | 오후 1시 자동 생성 여부 |
| status | enum | NOT NULL, default: 'pending' | 세션 상태 |
| notes | text | NULLABLE | 세션 메모 (강사용) |
| createdAt | timestamp | default: now() | 생성일 |
| updatedAt | timestamp | default: now() | 수정일 |

**status enum:**
```
pending        - 컨디션 체크 대기
generated      - 시퀀스 생성 완료
auto_generated - 오후 1시 자동 생성됨 (컨디션 없이)
modified       - 강사가 시퀀스 수정함
completed      - 수업 완료
cancelled      - 취소
```

## 2. ER 다이어그램

```
users (1) ─────── (1) members
  │                     │
  │                     ├── (0..N) weeklySchedules
  │                     │
  │                     └── (0..N) sessions ──── (0..1) exerciseSequences
  │
  ├── instructorId ──── (0..N) sessions (NULLABLE)
  └── instructorId ──── (0..N) weeklySchedules

exerciseCatalog (독립 참조 테이블)
```

### 3.4 컨디션 체크 (Condition)

#### `POST /api/condition/analyze`
컨디션 분석 (회원 전용, 사진 전송 → AI 분석 결과 반환)
```
Headers: Authorization: Bearer {accessToken}
Content-Type: multipart/form-data

Body:
  - image: (binary, JPEG/PNG) - 촬영한 사진

처리:
  1. 토큰에서 회원 ID 확인 (본인 확인 없음, 로그인 기반)
  2. Claude Vision API 호출: 사진 → 컨디션 분석
  3. AI 분석 결과 반환 (아직 DB 저장 안 함)

Response: 200
{
  "condition": {
    "energy": { "level": 6, "confidence": 0.85 },
    "mood": { "value": "CALM", "confidence": 0.90 },
    "stress": { "level": 3, "confidence": 0.80 },
    "sleep": { "quality": "FAIR", "confidence": 0.75 },
    "facialTension": {
      "forehead": { "level": 2, "confidence": 0.70 },
      "jaw": { "level": 3, "confidence": 0.65 },
      "asymmetry": { "value": "NONE", "confidence": 0.60 }
    },
    "swelling": { "level": "NONE", "confidence": 0.70 },
    "skin": {
      "pallor": { "value": false, "confidence": 0.50 },
      "flushed": { "value": false, "confidence": 0.50 }
    },
    "summary": "차분하고 안정된 상태이나 수면이 다소 부족해 보임",
    "exerciseNote": "급격한 고강도 운동보다는 점진적 강도 증가 권장"
  }
}

Errors:
  400 - 이미지에서 얼굴을 찾을 수 없음
  400 - 컨디션 체크 마감 (오후 1시 이후)
  429 - 요청 제한 초과
```

#### `POST /api/condition/register`
컨디션 확정 등록 (회원이 수정한 최종 컨디션 저장)

> **memberId 자동 조회**: `memberId`를 전달하지 않으면 로그인 사용자의 `members` 레코드를 자동 조회하여 사용. 회원이 직접 컨디션 등록 시 `memberId` 전달 불필요.

```
Request:
{
  "date": "2026-03-20",
  "memberId": "uuid (선택, 미전달 시 로그인 사용자의 members 레코드 자동 조회)",
  "conditionAiRaw": { ... },     // AI 원본 분석 결과
  "conditionFinal": {            // 회원 수정 후 최종
    "energy": 7,
    "mood": "CALM",
    "stress": 3,
    "sleep": "FAIR",
    "facialTension": { "forehead": 2, "jaw": 3, "asymmetry": "NONE" },
    "swelling": "NONE",
    "summary": "차분하고 안정된 상태이나 수면이 다소 부족해 보임"
  },
  "memberNote": "오른쪽 어깨가 좀 뻐근해요",
  "requestedCategories": ["core", "flexibility"]
}

Response: 201
{
  "sessionId": "uuid",
  "message": "컨디션이 등록되었습니다. 시퀀스 생성이 시작됩니다."
}
```

### 3.6 운동 시퀀스 (Sequences)

`POST /api/sequences/generate` - 시퀀스 생성
`PUT /api/sequences/:id` - 시퀀스 수정 (강사, wasModified 플래그 자동 설정)
`DELETE /api/sequences/:id/exercises/:order` - 운동 삭제 + 자동 reorder (강사)

> **시퀀스 재생성 시 기존 삭제**: `POST /api/sequences/generate`에서 같은 세션의 기존 시퀀스가 있으면 삭제 후 새로 생성. 오늘 탭에서 항상 최신 시퀀스만 표시.

> **회원별 세션 시간 기준**: 기본 50분 (회원 프로필 sessionDurationMinutes로 조절). 워밍업 3개(~7분) + 메인 8~12개(~36분) + 쿨다운 3개(~7분). 세트 간 휴식 15초 포함, 동적 운동 수 조절.

> **산전/산후 반영**: 산전 회원은 prenatal 운동 자동 포함 + 고강도 제외, 산전 미체크 회원은 prenatal 자동 제외. 타겟 근육/avoidExercises 필터링 적용.

### 3.7 운동 카탈로그 (Exercises)

`GET /api/exercises` - 카탈로그 조회 (354개 운동, 22개 카테고리)

## 4. 에러 응답 형식

```json
{
  "error": {
    "code": "MEMBER_NOT_FOUND",
    "message": "해당 회원을 찾을 수 없습니다.",
    "statusCode": 404
  }
}
```

**공통 에러 코드:**
```
AUTH_REQUIRED          - 401 - 인증 필요
AUTH_INVALID_TOKEN     - 401 - 유효하지 않은 토큰
AUTH_EXPIRED_TOKEN     - 401 - 만료된 토큰
FORBIDDEN              - 403 - 권한 없음
NOT_FOUND              - 404 - 리소스 없음
VALIDATION_ERROR       - 400 - 요청 데이터 검증 실패
FACE_NOT_DETECTED      - 400 - 이미지에서 얼굴 미감지
CONDITION_CHECK_CLOSED - 400 - 컨디션 체크 마감 (오후 1시 이후)
SCHEDULE_CONFLICT      - 409 - 스케줄 중복
LLM_GENERATION_FAIL    - 500 - 시퀀스 생성 실패
RATE_LIMIT_EXCEEDED    - 429 - 요청 제한 초과
INTERNAL_ERROR         - 500 - 서버 내부 오류
```
